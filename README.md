## 🔷 SECTION 3 — Latest Features & Changelogs (2024–2025)

### **LineageOS**
- Hard at work since Android 15's release in September, adapting unique features to this new version. Android 15 introduced several complex changes under the hood, but due to previous efforts adapting to Android 12-14, they rebased onto Android 15's code-base faster than anticipated.
- Introduced two new apps: Twelve (replaces aging music app) and Camelot (PDF viewer).
- Security patches from March 2024 to November 2024 merged. SeedVault and Etar updated. WebView updated to Chromium 131.0.6778.200.
- Extract utilities completely reworked — optimized from ~180 seconds down to ~30 seconds, then rewritten in Python achieving 6 seconds — a 30x speed improvement!
- Now using decimal integer system matching Android's QPRs. LineageOS 22.1 is based on Android 15 QPR1.
- **Security Patch:** Android security patch level: 1 August 2025 (for Raspberry Pi builds; device-specific varies)

### **Evolution X**
- Successfully migrated to LineageOS base (major change from AOSP)
- Android 16 QPR2 with April 2026 security patch available
- Both Android 15.0 (Vanilla) and Android 16.0 (GApps) versions released
- WPA3 Wi-Fi connection issues identified and patched in 10.7 with separate boot image fix.
- Vanilla and GApps dual variant publishing continues
- ROM includes hardware-accelerated video encoders. Video recording works without problems (on certain builds)

### **DerpFest**
- Android 15.1 builds rolling out
- DerpLauncher receives praise — "please don't ever try to imitate Pixel feel."
- BlockP firewall app included
- Suggestion for adding File Manager and Web Browser noted — "ROM is mostly debloated."

### **crDroid**
- crDroid 11 based on Android 15 released
- Continued KernelSU integration in official builds
- Game Space improvements
- Nintendo Switch support maintained

### **GrapheneOS**
- Support for 10th gen Pixels (Pixel 10, 10 Pro, 10 Pro XL, 10 Pro Fold) no longer considered experimental.
- All Android 16 security patches from April 2026 through August 2026 Android Security Bulletins included in security preview releases.
- Partnership with Motorola announced in March 2026, with devices expected in Q4 2026 or Q1 2027.
- Hardware + software USB blocking functionality reimplemented.
- Moved infrastructure away from OVHcloud (French provider) over privacy concerns.

### **CalyxOS**
- Android 14/15 support for Pixel devices and Fairphone
- microG updates for better push notification reliability
- Datura Firewall improvements
- Aurora Store integration enhanced

### **PixelOS**
- Android 15 support rolling out
- Device support expanding slowly
- Maintaining clean Pixel aesthetic focus

## 📝 Final Summary: State of Custom ROMs in 2025

The custom ROM scene in 2025 is **mature but increasingly specialized**. We're no longer in the Wild West of CyanogenMod days where ROMs competed on raw feature counts. Instead, we see clear philosophical divisions:

**Security-first ROMs** (GrapheneOS, CalyxOS) have professionalized, with GrapheneOS even pursuing partnership with Motorola for official device support in Q4 2026/Q1 2027. This represents a legitimization of privacy-focused mobile computing.

**Customization ROMs** (Evolution X, DerpFest, crDroid) continue to thrive for enthusiasts, but face the same challenge: **maintainer burnout**. As one GSI developer noted: "It seems like you can't trust users in the Android scene... How many people wanted to continue the legacy? Zero. Only feature and bug requests."

**The elephant in the room** is device support. Budget devices like your Samsung Galaxy M04 are increasingly orphaned. Developers only buy VFM mobiles in 20k budget category. Low range, high range mobiles get little to no development.

The future? Expect **consolidation** around a few well-maintained projects. GrapheneOS's Motorola partnership signals potential mainstream adoption. LineageOS remains the backbone. But for non-Pixel, non-flagship devices, the ROM scene is shrinking rather than growing.

**For Samsung Galaxy M04 specifically:** I must be honest — options are extremely limited. best path is likely:
1. Root with Magisk via Odin method (if bootloader unlockable)
